Procedural vs Photorealistic Floors

Can we get enough character without loading textures at all?

Procedural advantages

  • Zero loading — instantaneous
  • Seeded from board seed — deterministic, shareable
  • Scale-independent — no texture tiling seams
  • Full parameter control (line width, spacing, wear)
  • Cluster type is a shader param, not a texture swap
  • No NAS / asset dependency in production
  • Consistent with machine-world aesthetic

Procedural trade-offs

  • More shader code to write and maintain
  • Microdetail (scratches, dust) harder to get right
  • Requires GLSL knowledge or noise math
  • Can look too clean / synthetic if not tuned
Recommended hybrid: Procedural structure (grid lines, panel seams, Voronoi cells) + a single tiny noise texture (~64×64) baked from the board seed for microdetail variation. Best of both — no loading, full control, natural imperfection.
A

Full Procedural

GLSL ShaderMaterial generates all floor patterns. Metal grid, concrete panels, gravel scatter — all in the fragment shader. No texture files at all.

B

Procedural + Noise Texture

Procedural structure + one tiny (~64×64) seeded noise texture for surface microdetail. Feels imperfect and organic without loading heavy assets.

C

Keep Photorealistic

Use AmbientCG textures from the library. More work to tune but richer surface character out of the box.
