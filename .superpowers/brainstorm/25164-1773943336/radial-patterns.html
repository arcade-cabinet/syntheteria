Unified Radial Menu — Core Interaction Patterns

Three modes the same component needs to support. Which feels right?

Unit Context

Single-select ring. Center = faction color + Mark level + curved class/spec text. Outer ring = available actions. Lines connect center to options.

Tech Tree (Hub/Spoke)

Multi-select with LINE connections. Researched = filled green spokes. Available = dashed branches off discovered nodes. Locked = dim dashed from center. Ship-wheel pattern.

Fabrication (Nested Select)

Single-select with drill-down. Center = building. Ring 1 = robot classes. Selecting a class (SCOUT highlighted) fans out Ring 2 = specialization tracks. Two-level deep.

One Component, Three Modes

All three patterns share the same primitives: center disc, connection lines, outer circles, and curved text. The differences are in data and interaction mode:

  • Unit Context — single-select ring, action callbacks, static layout
  • Tech Tree — multi-select hub/spoke, pre-selected (discovered) nodes, branching sub-nodes
  • Fabrication — single-select with drill-down, ring 2 fans from selected ring 1 item
